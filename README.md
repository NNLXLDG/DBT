# SQL Server 数据库课程笔记

本仓库包含了 SQL Server 数据库课程的学习笔记，涵盖了数据库系统基础、SQL 语言、程序设计及系统设计等核心内容。

## 仓库要点大纲

### 第1章 数据库技术引入
- 数据库基本概念：数据(Data)、数据库(Database)、数据库管理系统(DBMS)、数据库系统(DBS)
- DBMS 的主要功能：数据定义功能(DDL)、数据组织与存储管理、数据操纵功能(DML)、事务管理与运行管理、数据库建立与维护

### 第2章 关系数据库标准语言 SQL
- 关系的性质：列同质性、元组原子性、行列无序性、行唯一性
- 关系模式与关系：结构描述与具体实例的区别
- 关系数据模型三要素：数据结构（二维表）、数据操作、完整性约束
- 完整性约束：实体完整性(Primary Key)、参照完整性(Foreign Key)、用户自定义完整性(Check/NotNull)

### 第3章 SQL 查询管理
- SQL 功能分类：数据定义(CREATE/DROP/ALTER)、数据查询(SELECT)、数据操纵(INSERT/UPDATE/DELETE)、数据控制(GRANT/REVOKE)
- SELECT 查询详解：去重(DISTINCT)、排序(ORDER BY)、分组(GROUP BY/HAVING)、多表连接(JOIN)、结果集限制(TOP N)

### 第4章 视图管理与 SQL 数据控制
- 视图的概念与作用：虚表定义、简化操作、增强安全性、逻辑独立性
- 视图的分类：普通视图、索引视图、分区视图
- 视图的定义与维护：CREATE VIEW 的语法规则、WITH CHECK OPTION 约束

### 第5章 Transact-SQL 程序设计
- T-SQL 基础：变量定义（局部变量 @，全局变量 @@）、DECLARE 赋值语句
- 语言元素：注释规范、常用系统内置函数（字符函数、数学函数等）

### 第6章 数据库恢复技术
- 事务(Transaction)概念：不可分割的逻辑工作单位
- 事务的 ACID 特性：原子性(Atomicity)、一致性(Consistency)、隔离性(Isolation)、持久性(Durability)
- 事务的状态：活动、部分提交、失败、异常结束、提交

### 第7章 数据库系统设计
- 设计内容：结构特性设计（静态模型）、行为特性设计（动态模型）
- 设计过程：规划阶段、需求收集分析、概念设计、逻辑设计、物理设计、数据库实施、数据库运行与维护
